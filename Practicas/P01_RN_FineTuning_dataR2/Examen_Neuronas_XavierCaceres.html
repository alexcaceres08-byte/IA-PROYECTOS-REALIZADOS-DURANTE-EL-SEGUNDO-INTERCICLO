Fundamentos de Machine Learning y Redes Neuronales

1. Qu es Machine Learning (Aprendizaje Automtico)?

El Machine Learning es una rama de la Inteligencia Artificial que permite a las computadoras aprender de los datos y mejorar su rendimiento sin ser programadas explcitamente para cada regla.

En lugar de escribir cdigo que diga: "Si la glucosa es > 100, entonces es paciente", alimentamos al modelo con miles de ejemplos mdicos y dejamos que el algoritmo encuentre ese patrn por s mismo.

Tipos Principales de Aprendizaje

  • Aprendizaje Supervisado (Supervised Learning): El modelo se entrena con datos que ya tienen la respuesta correcta (etiquetas).
    • Ejemplo: Tu dataset dataR2, donde ya sabamos quin era "Sano" (1) y quin "Paciente" (0).
  • Aprendizaje No Supervisado: El modelo busca patrones en datos sin etiquetas (agrupamiento o clustering).
  • Aprendizaje por Refuerzo: El modelo aprende a base de prueba y error (premios y castigos).

2. El Flujo de Trabajo en Ciencia de Datos

Para que un modelo funcione, seguimos una "Pipeline" estricta:

  1. Preprocesamiento (Preprocessing): Limpieza de datos. Las mquinas entienden nmeros, no texto.
    • Imputacin: Rellenar datos faltantes.
    • Escalado (Scaling): Poner todas las variables (Edad, Insulina, Glucosa) en la misma escala (ej. Standard Scaler) para que una variable grande no domine a las pequeas.
  2. Divisin de Datos (Train/Test Split):
    • Train (80%): Datos para que el modelo estudie.
    • Test (20%): Datos que el modelo nunca ha visto, usados para el examen final.
  3. Entrenamiento (Fit): El proceso matemtico de ajustar los pesos.
  4. Prediccin y Evaluacin: Medir qu tan bien funciona el modelo con mtricas matemticas.

3. Redes Neuronales Artificiales (ANN)

Son modelos inspirados en el cerebro humano, diseados para reconocer patrones complejos.

Arquitectura Bsica

  1. Capa de Entrada (Input Layer): Recibe los datos crudos. Tiene tantas neuronas como variables tengas (en tu caso, 9 variables: Age, BMI, Glucose, etc.).
  2. Capas Ocultas (Hidden Layers): Donde ocurre la "magia". Procesan la informacin combinando las entradas.
    • Dense Layer: Cada neurona se conecta con todas las de la capa anterior.
  3. Capa de Salida (Output Layer): Entrega el resultado final.
    • Para clasificacin binaria (Sano vs Paciente), suele tener 1 sola neurona.

Conceptos Clave del Entrenamiento

  • Neurona: Unidad bsica de procesamiento.
  • Pesos (Weights): La importancia que la red le da a cada conexin. Si el "BMI" es muy importante para el diagnstico, su peso ser mayor.
  • Sesgo (Bias): Un valor extra que ayuda a la neurona a activarse (similar a la ordenada al origen en una recta).

4. Funciones de Activacin

Deciden si una neurona debe "dispararse" o no. Sin ellas, la red sera una simple multiplicacin lineal.

  • ReLU (Rectified Linear Unit): Se usa en capas ocultas. Convierte los valores negativos a cero y deja pasar los positivos. Es muy rpida y eficiente.
  • Sigmoid: Se usa en la capa de salida para problemas binarios. Aplasta cualquier nmero para que quede entre 0 y 1.
    • Resultado cercano a 0 $\rightarrow$ Clase 0 (Paciente).
    • Resultado cercano a 1 $\rightarrow$ Clase 1 (Sano).

5. Optimizacin y Control de Errores

Cmo aprende la red? (Backpropagation)

La red hace una prediccin, compara con la realidad, calcula el error y "retrocede" ajustando los pesos para equivocarse menos la prxima vez.

  • Loss Function (Funcin de Prdida): Mide el error. En tu caso usamos binary_crossentropy (ideal para S/No).
  • Optimizer: El algoritmo que decide cmo ajustar los pesos.
    • Adam: Muy popular, ajusta la velocidad de aprendizaje automticamente.
    • RMSprop: Bueno para datos ruidosos.
  • Epochs (pocas): Cuntas veces la red ve el dataset completo. (Pocas pocas = no aprende; muchas pocas = memoriza).
  • Batch Size: Cuntos datos procesa antes de actualizar sus pesos.

Evitando el "Memorismo" (Overfitting)

Si la red se aprende los datos de memoria, fallar con pacientes nuevos.

  • Dropout: Es una tcnica donde "apagamos" aleatoriamente un porcentaje de neuronas (ej. 35%) en cada paso. Esto obliga a la red a no depender de neuronas especficas y ser ms robusta.

6. Mtricas de Evaluacin

  • Accuracy (Exactitud): Porcentaje total de aciertos. (ej. 80% significa que de 100 pacientes, diagnostic bien a 80).
  • Precision: De todos los que la red dijo "Es Sano", cuntos lo eran realmente? (Evita Falsos Positivos).
  • Recall (Sensibilidad): De todos los que realmente eran "Sanos", a cuntos detect la red? (Evita Falsos Negativos).
  • F1-Score: Un promedio armnico entre Precision y Recall. Es la mejor mtrica si tienes datos desbalanceados.
  • Matriz de Confusin: Tabla que muestra:
    • Verdaderos Positivos (TP)
    • Verdaderos Negativos (TN)
    • Falsos Positivos (FP - Error Tipo I)
    • Falsos Negativos (FN - Error Tipo II)

Parmetros y Hiperparmetros de una Red Neuronal

En el diseo de redes neuronales, es fundamental distinguir entre los valores que la red "aprende" sola y los que nosotros configuramos manualmente.

1. Parmetros Internos (Entrenables) Son los valores que la red ajusta automticamente durante el entrenamiento para minimizar el error (a travs del algoritmo de Backpropagation).

  • Pesos (Weights): Determinan la fuerza o importancia de la conexin entre neuronas. Si un biomarcador es crtico para detectar la enfermedad, la red le asignar un peso mayor.
  • Sesgos (Biases): Funcionan como un umbral de activacin, permitiendo que la neurona dispare una seal incluso si las entradas son bajas, ajustando la flexibilidad del modelo.

2. Hiperparmetros (Configurables) Son las variables que nosotros definimos antes de iniciar el entrenamiento. Su correcta eleccin (optimizada en nuestro caso mediante Grid Search) es vital para el xito del modelo.

  • Arquitectura: La estructura del "cerebro", definida por el nmero de capas ocultas y la cantidad de neuronas por capa (en nuestro modelo optimizado: 64 y 4 neuronas).
  • Funciones de Activacin: Reglas matemticas que procesan la informacin. Usamos ReLU en capas internas por su eficiencia y Sigmoid en la salida para obtener una probabilidad binaria (0 a 1).
  • Configuracin del Entrenamiento:
    • pocas (75): Cantidad de veces que la red repasa el dataset completo.
    • Batch Size (10): Frecuencia con la que se actualizan los pesos internamente.
    • Dropout (35%): Tcnica de regularizacin que "apaga" neuronas al azar para evitar que el modelo memorice los datos, forzndolo a aprender patrones generales.

Fase 3: Optimizacin del modelo de red neuronal (Fine Tuning)

Ajuste Fino (Fine Tuning) y Estrategia de Optimizacin

Tras establecer una lnea base con el modelo inicial, entramos en la etapa crtica del desarrollo: la optimizacin de hiperparmetros. El objetivo no es solo lograr que el modelo "aprenda", sino que generalice de manera ptima ante nuevos casos mdicos, maximizando la precisin y minimizando los falsos negativos.

Para lograr esto, abandonamos el enfoque de prueba y error manual y adoptamos una metodologa cientfica y exhaustiva:

1. Bsqueda Sistemtica (Grid Search)

Utilizamos la tcnica de Grid Search, que explora exhaustivamente una "malla" de combinaciones posibles de parmetros. En lugar de adivinar qu configuracin funciona mejor, el algoritmo entrena y evala el modelo para cada combinacin definida, permitindonos encontrar matemticamente la arquitectura ganadora.

2. Validacin Cruzada (K-Fold Cross-Validation)

Para asegurar que los resultados no sean producto de la suerte (sesgo por una particin especfica de datos), implementamos una validacin cruzada con $K=5$ folds.

  • Esto significa que cada configuracin se entrena y prueba 5 veces en subconjuntos de datos diferentes rotativos.
  • El resultado final es el promedio de estos 5 experimentos, ofreciendo una mtrica de rendimiento mucho ms robusta y confiable.

3. Parmetros a Optimizar

El proceso se divide en dos etapas lgicas:

  • Paso A - Dinmica de Entrenamiento: Ajustamos cmo aprende la red (Optimizadores como Adam/RMSprop, cantidad de pocas y tamao del Batch).
  • Paso B - Arquitectura y Regularizacin: Ajustamos la capacidad del cerebro (cantidad de neuronas en capas ocultas) e introducimos Dropout, una tcnica de regularizacin vital que "apaga" neuronas aleatoriamente para prevenir el memorismo (overfitting) y forzar a la red a aprender patrones ms robustos.

            certeza = (prediccion - mina)/(maxa - mina)
            
        marcas.append(marca)
        certezas.append(str(int(certeza * 100)) + '%')
        
    return marcas, certezas
